## 🧠 层级一：显性记忆层 (Markdown 领域)

在开始任何对话或开发之前，你必须立刻阅读以下两个文件，它们包含了本项目的“静态灵魂”与“动态状态”：

1. **📜 阅读 `ARCHITECTURE.md` (或类似指南)**：无论接手什么任务，绝对**不能**违反此处规定的技术栈、端口和代码范式。
2. **🎯 阅读 `.evo-lite/active_context.md`**：这是本项目的**当前进行时单据**。上一次会话结束时，进度停在了哪里？有什么未解的 Bug？下一步该干什么？全在这里。*(读完它，你就无缝接过了上个 AI 的上下文)*

## 🗄️ 层级二：隐性记忆层 (Vector RAG 领域) & `mem` CLI 互动

我们内置了一个随 Git 走的小型脱机向量数据库 (`.evo-lite/memory.db`)。当你遇到解决不了的奇葩报错、接手看不懂的架构链时，不要凭空猜测，你**必须优先依赖底层 CLI 工具**进行探索：

1. **探索能力**: 运行 `./.evo-lite/mem help` (Windows 环境为 `.\.evo-lite\mem.cmd`) 可查看所有可用记忆与清洗指令（如 `stats`, `compact`, `forget`, `verify`, `export`, `import` 等）。
2. **查阅历史血泪史**:
   ```bash
   ./.evo-lite/mem recall "报错信息或设计疑问"
   ```
   阅读终端返回的结果，里面往往藏着秒杀当前问题的宝贵资产。

## 🔄 核心使命：无缝交接规约 (Handover Protocol)

由于你无法预知人类何时结束会话，因此你的强制触发器是：**每当完成一个独立的功能闭环、修好一个 Bug、或执行完 `git commit/push` 操作后，必须立刻主动执行交接，严禁“聊完就跑”。**

### 1. 覆写显性状态 (状态机流转)
使用你的文件修改工具，**覆写**本项目的 `.evo-lite/active_context.md`：
- 将刚做完的事项追加入 `## 2. 🚧 当前进度与任务` 列表并正确打上 `[x]`。
- 移除已完成的“下一步计划”，并为下任接管的 AI 明确写出最新的下一步指引。

### 2. 打入隐性碎片 (沉淀离线经验池)
当你解决了一个隐蔽Bug或完成了架构级重构，**必须**将其永久存入向量库：
```bash
# 格式要求：内容必须用双引号包围，简明扼要地写明“问题-原因-解法及提交溯源”
./.evo-lite/mem remember "遇到了 [某特性缺陷]，原因是 [某底层机制]，解法为 [应用了某段代码] (溯源历史点: [Commit: xxx])"
```
*(注：底层钩子会在你执行完毕时打印终端高亮提醒。若格式违规，底层脚本将抛出中文致命异常报错，请仔细阅读红/黄字警告并重试！)*

## ⚠️ 记忆蒸馏法则 (Memory Distillation Rules)

为防止劣质数据污染 RAG 缓存，上述的 `mem remember` 动作必须遵守以下铁律（底层 Node.js 脚本已植入正则断点拦截）：

1. **质量与容量防呆**: 字符过少的流水账将被拒收。当库内承载达到系统阈值（约15条）后，底层脚本将抛出锁库告警，生成 `.evo-lite/MEMORIES_TO_COMPACT.md` 唤醒协议，强制由你代为进行高维、深度的中文摘要提炼 (`compact` 协议)。
2. **高维精简思路**: 仅记录重要的跨文件契约、绕过级重构、防呆思路变迁。
3. **强制并精确溯源**:
   ```text
   1. **核心主题词 A**: 技术规约。(溯源历史点: [Commit: aaa111, bbb222])
   2. **核心契约词 B**: 跨模块防御机制。(溯源历史点: [Commit: ccc333])
   ```
   🚨 **警告**：绝不允许在输入总结时使用省略号 `(...)` 代表代码段，或遗漏溯源哈希！所有的 `[Commit: xxx]` 必须精确且独立存在。

作为专属于本项目的 Evo-Lite 助手，你必须遵守以下纪律：

   永远提供最简单直接的解答。禁止过度设计，禁止脱离 Node.js / Python 原生特性凭空造框架。
