## 实现细节 (Implementation)
[bug-fix]
CodePLC (registered hive child, no templates/ tree) dogfooded the 2026-07-20 nurture and hit two classes of failure inside runChildRuntimeTests, which the comment at its definition promises is 'safe to run inside a child hive: they build their own temp mother/child fixtures and never touch templates/'. That promise was broken: (1) T-hive-version-truth, T-nurture-preserves-manifest, T-nurture-transactional used motherRoot=process.cwd() + required its package.json -- in mother mode process.cwd() happens to be the mother root so this worked by accident, but in a real child, cwd is the CHILD's own project root (no evo-lite package.json guarantee), throwing MODULE_NOT_FOUND before any assertion ran. Fixed by giving each a synthetic temp mother dir with a literal package.json version, matching the pattern already used by T-hive-status. (2) Every test from T-spec-cli through EOF (~1000 lines: spec-portfolio, code-perception, mcp-server, sync-runtime, template-manifest contract tests) resolves modules via TEMPLATE_CLI_DIR = <workspaceRoot>/templates/cli, which only exists in a mother checkout -- these were appended after the runChildRuntimeTests boundary without re-checking the child-safety invariant, so they threw MODULE_NOT_FOUND in a real child too. Fixed with a single IS_CHILD_RUNTIME early-return guard placed right before T-spec-cli: the function is invoked twice (child-only path skips T13-T27 and calls this; mother path calls this too after T13-T27), so the guard must check the flag rather than being unconditional. Verified two ways: (a) CodePLC's own environment (has its own local workaround patch, ran clean) and (b) a from-scratch simulation -- fresh temp dir containing only the freshly-synced .evo-lite/cli tree (no templates/ sibling, node_modules symlinked in) -- which exercises the MOTHER's actual fix and passed end-to-end, printing the new skip line and 'passed (child mode)'. Full mother-mode governance suite also green (no regression). This was surfaced by CodePLC's own agent while reviewing the earlier nurture; they patched their local child mirror as a stopgap. This commit is the canonical mother-side fix; a follow-up nurture --force will supersede their local patch with this one.
