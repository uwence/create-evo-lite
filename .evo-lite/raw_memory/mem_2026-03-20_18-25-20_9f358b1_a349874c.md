## 实现细节 (Implementation)
[ActiveContextBeforeRecall]
Clarified the debugging retrieval order so agents must inspect active_context.md before deciding to recall historical memories. Recall stays mandatory for unknown errors, repeated failures, complex dependency chains, or risky legacy mechanisms, but it now explicitly serves as the second layer after the current project state machine, not a substitute for it. This reduces duplicate work, keeps the current focus/backlog/trajectory in play, and makes the recall-first rule less likely to be misread as skipping the active context.
