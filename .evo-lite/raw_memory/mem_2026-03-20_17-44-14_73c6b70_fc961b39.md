## 实现细节 (Implementation)
[RerankerDegradePersistence]
Changed verify and model-loading behavior so reranker download failures no longer spam retries on every ordinary startup check. When the reranker cannot be fetched, Evo-Lite now records a degraded-state marker, continues in vector-only mode, and only retries when the user explicitly runs verify with --retry-reranker. In the same pass, verify now treats empty database files without raw_memory/chunks tables as fresh initialization state instead of reporting them as database corruption, and regression coverage was added for both behaviors.
