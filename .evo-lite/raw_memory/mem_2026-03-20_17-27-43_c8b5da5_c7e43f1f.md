## 实现细节 (Implementation)
[WorkspaceRootDiscipline]
Hardened the evo-lite protocol against agents inventing nested wrapper directories like project/, app/, or workspace/ inside an already-open project root. The rules now explicitly define the opened workspace as the single source of truth for root-relative workflow paths, and /evo startup guidance also warns that extra wrapper directories are forbidden unless the user explicitly requests a subproject or monorepo package. This prevents accidental breakage of .agents, .evo-lite, mem.cmd, and path-based workflow assumptions.
