# 📦 进度存档与交接协议 (/mem)

当你在本会话中完成了一个独立的功能点、修复了一个 Bug，或者你需要主动结束当前工作闭环时，**必须调用此协议**以确保上下文和经验被安全存档，供下一任 AI 助理（或者下一次你的唤醒）无缝接管。

步骤：
1. **覆写显性进度墙**: 请使用文件编辑工具修改本项目根目录下 `.evo-lite/active_context.md`。
   - 定位到 `## 2. 🚧 当前进度与任务` 列表，将你刚刚完成的任务标记为完成 `[x]`。如果还没有对应的待办，那就新增一行记录你的战果。
   - 清理旧的待办，并为你自己（或者接管的 AI）在 `## 3. 📝 下一步行动指南` 中明确写出下个阶段面临的任务盲区。
2. **经验向量记忆 (可选但强烈建议)**: 请主动回顾刚才的工作中，是否存在隐蔽的环境坑、复杂的跨组件通信流、或者是值得后来者查阅的设计决策。
   - 若有，请提炼总结，在终端静默运行（注意，**禁止流水账，必须严格附带 Commit 短哈希**）：
   ```bash
   .\.evo-lite\mem.cmd remember "这里写下简明的核心主题: 遇到了XX问题，原因是XX，最终利用XX机制解决。(溯源历史点: [Commit: $(git rev-parse --short HEAD)])"
   ```
   *(注: 如果非 Windows 平台，请使用 `./.evo-lite/mem` )*
3. **版本快照约束**: 确认所有的代码文件都已保存。通过终端对相关的代码变更执行标准的 Commit:
   ```bash
   git add .
   git commit -m "feat/fix: 简要描述变更"
   ```
4. **交接完成反馈**: 使用简炼的口语向开发者宣告：“交接协议已执行完毕。显性任务上下文已推演，经验记忆已入库。目前项目处于干净且随时可挂起的状态。”
