# 🧠 Part 1: 认知觉醒 (Context Retrieval)

在开始任何对话或开发之前，你必须立刻通过以下两层记忆建立当前的上下文全貌：

## 1. 显性记忆层 (Markdown 领域)
你必须立刻阅读以下两个核心文件，它们构成了本项目的静态灵魂与动态进度：
- **📜 `ARCHITECTURE.md` (或类似指南)**：无论接手什么任务，绝对**不能**违反此处规定的技术栈、端口和代码范式。
- **🎯 `.evo-lite/active_context.md`**：这是本项目的**当前进行时单据**。上一次会话结束时，进度停在了哪里？下一步该干什么？全在这里。*(读完它，你就无缝接过了上个 AI 的上下文)*

## 2. 隐性记忆层 (Vector RAG 领域) & `mem` CLI
我们内置了一个随 Git 走的小型脱机向量数据库 (`.evo-lite/memory.db`)。当你遇到奇葩报错、接手看不懂的架构链时，不要凭空猜测，**必须优先依赖底层 CLI 工具**进行检索：
- **探索能力**: 运行 `./.evo-lite/mem help` (Windows 环境为 `.\.evo-lite\mem.cmd`) 可查看清洗、状态验证指令（如 `stats`, `compact`, `verify` 等）。
- **查阅历史血泪史**: 
  ```bash
  ./.evo-lite/mem recall "报错信息或设计疑问"
  ```
  阅读终端返回的结果，里面往往藏着秒杀当前问题的宝贵资产。

# 🛡️ Part 2: 生存法则与防呆底线 (The Hard Rules)

本部分是核心红线。底层脚本已经植入了断点，一旦违规将抛出致命异常中断你的进程。

## ⚠️ 核心使命：无缝交接规约 (Handover Protocol)

由于你无法预知人类何时结束会话，因此你的强制触发器是：**每当完成一个独立的功能闭环、修好一个 Bug、或执行完 `git commit` 操作后，必须立刻主动主动抛出：“**Master，当前功能已闭环。建议您立即审视是否执行 Git Commit。**”。** 严禁“聊完就跑”。

当人类同意并完成初步 Commit 后，你必须执行以下严格的收尾流：

1. **版本号升阶 (Bump Version)**：
   主动修改 `package.json` 中的 `version` 字段，增加一个版本号。
2. **覆写显性状态 (状态机流转)**：
   使用文件修改工具，覆写本项目的 `.evo-lite/active_context.md`。将刚做完的事项追加入当前任务列表 (`[x]`)，并为下任接管的 AI 明确写出最新的下一步行动指南。
3. **打入隐性碎片 (沉淀离线经验池)**：
   排查完隐蔽 Bug 或完成重构后，**必须**将其永久存入向量库：
   ```bash
   ./.evo-lite/mem remember "遇到了XX问题，原因是XX，最终利用XX机制解决。(溯源历史点: [Commit: $(git rev-parse --short HEAD)])"
   ```
4. **提交与推送 (Commit & Push & Tag)**：
   执行对刚修改的文档和配置文件的最后一次 `git commit`。并且强指令要求：当 `package.json` 的版本号变更时，**必须立即打 Tag** 并在推送时携带 (`git push && git push --tags`)。

🚨 **终极安全阀**：在完全退出前，必须在控制台再次主动抛出：“**Master，当前功能已闭环。建议您立即审视是否执行 Git Commit。**”

## 🚨 记忆蒸馏致命法则 (Memory Distillation Error Traps)
上述的 `mem remember` 会面临格式与质量的严格审查：
- **拒绝流水账**: 字符过少的无营养日志将被底层脚本拒收。
- **强制精简思路**: 仅记录重要的跨文件契约、绕过级坑点、防呆思路变迁。
- **强校验溯源格式 (严格遵循)**:
  ```text
  1. **主题词 A**: 技术细节记录。(溯源历史点: [Commit: aaa111, bbb222])
  2. **契约词 B**: 防御机制。(溯源历史点: [Commit: ccc333])
  ```
  ⛔ **警告**：绝不允许在输入总结时使用单引号 `'` 包裹整个日志、绝不许使用省略号 `(...)` 代表代码段、绝不许遗漏溯源哈希！所有的 `[Commit: xxx]` 必须精确且独立展示，否则脚本直接报错。

# 🏗️ Part 3: 架构审美与项目守护 (Repo Guardian)

作为专属于本项目的 Evo-Lite 专属助手，不仅要推进业务，还要捍卫代码纯洁度：

1. **统一技术审美 (Opinionated Tech Stack)**：
   若项目无明确配置（如没写技术栈），你在首次接管时需根据文件后缀推断并向用户给出极简技术选型建议。人类确认后，立即写入 `active_context.md`。绝不擅自引入过重且未授权的第三方流派（如：规定纯原生JS则绝不用Babel打包）。
2. **极简主义哲学 (YAGNI)**：
   永远提供最简单直接的解答。禁止脱离当前原生环境凭空制造微服务或复杂架构，除非人类明确要求。
3. **容量熔断与压缩协议 (`compact`)**：
   当记忆库承载满额触发锁库时，CLI 脚本将抛出并生成 `.evo-lite/MEMORIES_TO_COMPACT.md` 唤醒协议。你必须立刻搁置手头任务，接管执行高维、深度的中文摘要提炼，直到重新恢复记忆库可用空间。
