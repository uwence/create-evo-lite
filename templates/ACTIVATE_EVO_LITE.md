# 🚀 激活 Evo-Lite 极简核心流 (ACTIVATE_EVO_LITE)

你好，AI 助手！当你读取这个文件时，说明我们正在一个**由 Evo-Lite (Serverless Memory Bank) 赋能**的项目中。
本项目采用了极其轻量级的纯本地“显隐双层记忆”架构。**我是你的记忆外挂，以下是你必须遵守的生存法则。**

---

## 🧠 层级一：显性记忆层 (Markdown 领域)

在开始任何对话或开发之前，你必须立刻阅读：
1. **📜 `ARCHITECTURE.md` (或类似指南)**：本项目的核心规范与技术栈，绝对禁止违反。
2. **🎯 `.evo-lite/active_context.md`**：本项目的**当前进行时单据**。上一次进度卡在哪里？下一步该干什么？全在这里。

---

## 🗄️ 层级二：隐性记忆层 (Vector RAG 领域) & `mem` CLI 互动指南

我们内置了一个强大的脱机向量记忆库 (`.evo-lite/memory.db`)。**本项目绝大多数的防呆、自检、洗盘与流程控制钩子全部埋在底层脚本中，触发它们的唯一方式就是呼叫我们的内置 CLI！**

请时刻在内置终端 (Terminal) 中主动调用：

1. **探索能力**: 运行 `./.evo-lite/mem help` 可查看所有可用记忆与清洗指令（如 `stats`, `compact`, `forget`, `verify` 等）。
2. **查阅历史血泪史**: 遇到不明 Bug 或接手看不懂的架构链时，先检索记忆库：
   ```bash
   ./.evo-lite/mem recall "报错信息或设计疑问"
   ```
3. **沉淀闭环经验**: 当你解决了一个隐蔽 Bug 或完成了高维重构，**必须**主动将经验打入向量碎层：
   ```bash
   ./.evo-lite/mem remember "遇到了 [某缺陷]，原因是 [某机制]，解法为 [某代码] (溯源历史点: [Commit: xxx])"
   ```
   *(注: 格式必须包含问题-原因-解法及提交溯源。任何违规写法都会触发底层的中文致命异常报错，请仔细阅读终端抛出的红/黄字提示并根据其指导修正指令)*。

## 🔄 核心使命：无缝交接规约 (Handover Protocol)

由于你无法预知人类何时结束会话，因此你的强制触发器是：**每当完成一个独立的功能闭环、修好一个 Bug、或执行完 `git commit/push` 操作后，必须立刻同步执行交接**：

1. **覆写显性状态**: 强制调用文件修改工具更迭 `.evo-lite/active_context.md` 源文件:
   - 将刚做完的事项追加入 `## 2. 🚧 当前进度与任务` 列表并正确打上 `[x]`。
   - 移除已完成的“下一步计划”，并为下任接管的 AI 明确写出最新的下一步指引。
2. **打入隐性碎片**: 确保如上所述的 `mem remember` 已被合理调用。*(底层钩子会在你执行完毕时打印终端高亮提醒)*

## 🛡️ 层级三：架构铁律与归档

1. **统一审美规范 (Opinionated Tech Stack)**：若项目中无明确配置，请在首次回复时向用户提出符合本项目极简理念的技术选型，经人类确认后立刻将其写入 `active_context.md`。
2. **极简主义 (YAGNI)**：永远提供最直达的解法，禁止过度设计与抽象，禁止引入非必要或过重的第三方依赖。
3. **自动建库流闭环 (Release Protocol)**：当 `package.json` 中的 `version` 字段发生变更时，**必须立即打 Tag 并推送** (`git push && git push --tags`)。
